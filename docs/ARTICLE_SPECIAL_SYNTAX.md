# Sintaxe Especial para Componentes Ricos

Além da sintaxe básica (##, ###, -, **), você pode usar estas sintaxes para componentes avançados:

## 📦 Key Takeaways (Resumo Executivo)

```
[TAKEAWAYS: Título do Resumo]
- **Item 1:** Descrição do primeiro ponto-chave
- **Item 2:** Descrição do segundo ponto-chave
- **Item 3:** Descrição do terceiro ponto-chave
[/TAKEAWAYS]
```

## 🎯 Blueprint (Framework Numerado)

```
[BLUEPRINT: Título do Framework]
1. **Etapa 1:** Descrição da primeira etapa
2. **Etapa 2:** Descrição da segunda etapa  
3. **Etapa 3:** Descrição da terceira etapa (destacada)
[/BLUEPRINT]
```

## 🛠️ Tech Stack

```
[STACK: Título da Stack]
- **CRM Core:** Salesforce / HubSpot
- **Analytics:** Tableau / Metabase
- **Automation:** Zapier / n8n
[/STACK]
```

## 🚩 Red Flags (Alertas)

```
[REDFLAGS: Sinais de Alerta]
- Primeiro sinal de problema
- Segundo sinal de problema
- Terceiro sinal de problema
[/REDFLAGS]
```

## 📋 Steps Grid (Passos de Ação)

```
[STEPS: Protocolo de Implementação]
1. **Passo 1:** Descrição da primeira ação
2. **Passo 2:** Descrição da segunda ação (destacada)
3. **Passo 3:** Descrição da terceira ação
[/STEPS]
```

## 💡 Info Box (Nota Destacada)

```
[INFO]
**Dica Importante:** Texto da dica ou nota relevante.
[/INFO]
```

---

## Exemplo Completo de Artigo:

```
## Por que RevOps é Crítico?

RevOps não é um departamento. É uma **arquitetura de receita**.

[TAKEAWAYS: Executive Summary]
- **Alinhamento:** Marketing, Vendas e CS falam a mesma língua de métricas
- **Eficiência:** Redução de 40% no CAC através de automação inteligente
- **Previsibilidade:** Forecast com precisão acima de 90%
[/TAKEAWAYS]

### O Problema Estrutural

A maioria das empresas opera em silos. Marketing gera MQLs que Vendas ignora. Vendas fecha contratos que CS não consegue onboar.

[REDFLAGS: Sinais de Colapso]
- Taxa de conversão MQL→SQL abaixo de 15%
- Churn acima de 5% ao mês
- Vendedores gastando mais de 30% do tempo em tarefas manuais
[/REDFLAGS]

## Como Implementar RevOps

[BLUEPRINT: Framework de Implementação]
1. **Fase 1 - Auditoria:** Mapeie todos os sistemas e fluxos atuais
2. **Fase 2 - Unificação:** Integre CRM + Marketing Automation + CS Platform
3. **Fase 3 - Automação:** Elimine trabalho manual repetitivo
4. **Fase 4 - Dashboards:** Crie visibilidade única de métricas
[/BLUEPRINT]

[STACK: Ferramentas Essenciais]
- **CRM:** HubSpot / Salesforce
- **Automação:** Zapier / Make
- **BI:** Metabase / Looker
- **CS Platform:** Gainsight / ChurnZero
[/STACK]

[STEPS: Protocolo de 30 Dias]
1. **Semana 1:** Auditoria completa de sistemas e processos
2. **Semana 2:** Definição de métricas únicas (North Star)
3. **Semana 3:** Implementação de integrações críticas
4. **Semana 4:** Treinamento de times e go-live
[/STEPS]

[INFO]
**Atenção:** RevOps não é um projeto de TI. É uma transformação cultural que exige buy-in do C-Level.
[/INFO]

---

## Conclusão

RevOps é o sistema operacional da receita moderna.
```
